# Análisis del Reporte 19

| Key     | Value                    |
| ------- | ------------------------ |
| Archivo | Empresa.vb               |
| Método  | GeneraCalculos_Reporte19 |

## Propósito

Calcula para cada par `(padre, hijo)` del grupo económico la **participación directa e indirecta** del padre sobre el hijo, considerando todas las cadenas de propiedad posibles. Si la participación total supera el 50%, se registra en `TEMP_Control` como una relación de control.

---

## Estructura de datos clave

```
tEmpresaPath
├── Done: tStatus   (xOpen | xClose | xLoop)
└── Path(): Long[]  (cadena de IDs: [raíz, ..., nodo_actual])

TEMP_Participaciones
├── CodRemitente   → empresa poseída (la inversión / subsidiaria)
├── CodIntegrante  → empresa que tiene acciones en CodRemitente (el dueño)
└── Participacion  → % que CodIntegrante posee de CodRemitente
```

El árbol se expande **hacia abajo** (dueño → subsidiarias): la query `WHERE CodIntegrante = [nodo_actual]` devuelve los `CodRemitente` que el nodo actual posee.

---

## El algoritmo en fases

### Fase 0 — Setup

Limpia `TEMP_Control` y ejecuta el proceso completo **dos veces**: con `NUMERO_RAIZ = 1` (empresa raíz principal) y `NUMERO_RAIZ = 57` (otra entidad del grupo, probablemente la reportante).

### Fase 1 — Expansión del árbol (`ExplotaArbol`)

Expande desde la raíz **hacia abajo** todas las rutas posibles del grafo de propiedad hasta que no quedan nodos `xOpen`. Cada ruta acumula los IDs visitados en `Path()`. Al extender un nodo, si la subsidiaria encontrada ya está en el `Path()` actual → se marca como `xLoop` (ciclo detectado).

```
Raíz: [R]
→ [R, A]        xOpen   (R posee A)
→ [R, A, X]     xClose  (A posee X; X no posee nada)
→ [R, A, R]     xLoop   ← ciclo: R aparece dos veces
```

### Fase 2 — Resolución de ciclos (`RedistribuyeLoops`)

Si existe algún `xLoop`, el algoritmo **rompe el ciclo** eliminando la participación que lo cierra y redistribuye proporcionalmente las participaciones restantes en esa empresa para que sumen 100%. Repite hasta que no haya más ciclos.

**Escenario:** R posee el 100% de A; A, B y C son accionistas de R.

```
TEMP_Participaciones (estado inicial)

 CodRemitente  │ CodIntegrante │ Participacion
 (poseída)     │ (dueño)       │
───────────────┼───────────────┼──────────────
 A             │ R             │ 100%   R posee A
 R             │ A             │  50%   A posee 50% de R  ← ciclo
 R             │ B             │  30%   B posee 30% de R
 R             │ C             │  20%   C posee 20% de R
```

**Expansión:** `[R] → [R, A] → [R, A, R]` xLoop (R ya estaba en el path)

**`RedistribuyeLoops([R, A, R])`:**

```
xCodRemitente  = Path(último)     = R
xCodIntegrante = Path(penúltimo)  = A

1. DELETE WHERE CodRemitente=R AND CodIntegrante=A
   → Elimina la participación de A en R (50%)

2. SELECT Sum(Participacion) WHERE CodRemitente=R → 30% + 20% = 50%
   Factor = 100 / 50 = 2.0

3. UPDATE Participacion = Round(2.0 × Participacion, 2) WHERE CodRemitente=R
   → B: 30% × 2.0 = 60%
   → C: 20% × 2.0 = 40%
```

```
TEMP_Participaciones (después de la redistribución)

 CodRemitente  │ CodIntegrante │ Participacion
───────────────┼───────────────┼──────────────
 A             │ R             │ 100%
 R             │ B             │  60%   proporción B:C preservada (30:20 → 60:40)
 R             │ C             │  40%
```

Re-expansión: `[R] → [R, A] → [R, A]` xClose (A ya no posee R). Sin más ciclos.

### Fase 3 — Cálculo para cada par `(padre, hijo)`

Bucle O(n²) sobre todas las empresas del grupo:

**3a. `GetCoincidencias`** — De todas las rutas expandidas, extrae solo las que contienen la secuencia `padre → ... → hijo`, y las recorta a ese segmento. Elimina duplicados.

**3b. `OmiteSignificativas`** — Recorre cada ruta desde el inicio. Mientras el arco `X→Y` tenga participación > 50%, descarta a X y arranca desde Y. Razón regulatoria: una empresa con más del 50% de acciones sobre otra tiene control efectivo total (siempre ganará en votaciones), por lo que ese nodo es neutro para el cálculo y se elimina para no distorsionar el resultado.

**3c. `SimplificaRutas`** — Elimina rutas redundantes: si la ruta A es sufijo de la ruta B (B "contiene" a A al final), B se descarta. Solo se conservan los caminos más específicos.

**3d. `CalculaParticipacion`** — Para cada ruta simplificada, calcula el **producto de las participaciones** a lo largo de la cadena:

```
Ruta [padre → X → Y → hijo]:
  participacion = (padre%X / 100) × (X%Y / 100) × (Y%hijo / 100) × 100
```

Clasifica como **directa** si la ruta tiene exactamente 2 nodos (`[padre, hijo]`), o **indirecta** si tiene intermediarios.

**3e. Inserción en `TEMP_Control`** — Si `P_Directa + P_Indirecta > 50%`, registra el par.

## Diagrama de flujo

```
NUMERO_RAIZ = 1, luego 57
  │
  └─► ExplotaArbol(raíz)
        │
        └─► ¿hay xLoop?
              ├─ Sí → RedistribuyeLoops → volver a ExplotaArbol
              └─ No ─►
                    Para cada par (padre, hijo):
                      GetCoincidencias
                        → OmiteSignificativas
                          → SimplificaRutas
                            → CalculaParticipacion
                              → si total > 50%: INSERT TEMP_Control
```

## Decisiones de diseño notables

| Decisión                                    | Por qué                                                                                                                       |
| ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Doble pasada (raíz 1 y 57)                  | El reporte exige el punto de vista de dos entidades distintas del grupo                                                       |
| Redistribución proporcional de ciclos       | Evita participaciones > 100% sin descartar participaciones circulares                                                         |
| Umbral >50% (no ≥50%)                       | Define "control efectivo" — mayoría simple estricta                                                                           |
| `OmiteSignificativas`                       | Regla regulatoria: >50% equivale a control total; ese nodo es neutro y se omite para no distorsionar el producto de la cadena |
| `SimplificaRutas` (eliminar superconjuntos) | Un camino largo que contiene un camino corto al final ya está capturado por el corto                                          |

## Implicaciones para el port a .NET

El algoritmo es un **DFS con detección de ciclos + producto de probabilidades sobre grafos dirigidos**. En C# esto se traduce limpiamente a:

- Grafo representado como `Dictionary<long, List<(long subsidiaria, decimal porcentaje)>>` donde la clave es el dueño (CodIntegrante) y el valor lista sus empresas poseídas (CodRemitente)
- Rutas como `ImmutableList<long>` (eficiente para copiar con `Add` al expandir) — sin slot dummy en índice 0: el código VBA hacía `ReDim Path(N)` y escribía desde el índice 1, dejando `Path(0) = 0` sin uso; en .NET se usa indexación 0-based directamente
- La resolución de ciclos es una mutación del grafo antes de la expansión final
- El cálculo por par es paralelizable (`Parallel.ForEach` sobre el producto cartesiano)
